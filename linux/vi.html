Introduction to Shell

Shell Commands +

1. /bin/bash

The bash shell executable path. You can use this to start a new bash shell instance.

$ /bin/bash
# This starts a new bash shell

2. which

Locates a command's executable in your PATH.

$ which ls
/bin/ls

$ which python
/usr/bin/python

3. type operator

Shows information about a command, whether it's an alias, function, or executable.

$ type ls
ls is aliased to 'ls --color=auto'

$ type cd
cd is a shell builtin
Shell Variables +

1. Variable in the shell

Creating and using variables in the shell.

$ name="John"
$ echo $name
John

2. Variable in the quotes

Using variables within different types of quotes.

$ name="John"
$ echo "Hello, $name"
Hello, John

$ echo 'Hello, $name'
Hello, $name

3. $SHELL

Environment variable that contains the path to the current shell.

$ echo $SHELL
/bin/bash

4. $PATH

Environment variable that lists directories where executable programs are located.

$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin

  echo "Usage: $0 <source_directory>"
  exit 1
fi

BACKUP_DIR="/tmp/backup"

BACKUP_FILE="backup_$DATE.tar.gz"

  echo "Backup created successfully: $BACKUP_DIR/$BACKUP_FILE"
else
  echo "Backup failed!"
fi

Linux System Management

File Permissions +

1. R (Read)

Permission to read a file or list directory contents.

$ ls -l file.txt
-r--r--r-- 1 user group 1024 May 10 10:30 file.txt
# The 'r' in positions 1, 4, and 7 indicates read permission for owner, group, and others

2. W (Write)

Permission to modify a file or create/delete files in a directory.

$ ls -l file.txt
--w------- 1 user group 1024 May 10 10:30 file.txt
# The 'w' in position 2 indicates write permission for the owner only

3. X (Execute)

Permission to execute a file or access a directory.

$ ls -l script.sh
---x------ 1 user group 1024 May 10 10:30 script.sh
# The 'x' in position 3 indicates execute permission for the owner only

4. Owner User (u)

The user who owns the file or directory.

$ ls -l file.txt
-rw-r--r-- 1 john users 1024 May 10 10:30 file.txt
# 'john' is the owner user of this file

5. Owner Group (g)

The group that owns the file or directory.

$ ls -l file.txt
-rw-r--r-- 1 john users 1024 May 10 10:30 file.txt
# 'users' is the owner group of this file

6. Other (o)

All other users who are not the owner or in the owner group.

$ ls -l file.txt
-rw-r----- 1 john users 1024 May 10 10:30 file.txt
# The last three characters '---' show that others have no permissions

7. All (a)

Refers to all three categories: owner, group, and others.

$ chmod a+x script.sh
# Gives execute permission to owner, group, and others
$ ls -l script.sh
-rwxr-xr-x 1 john users 1024 May 10 10:30 script.sh

8. chmod (Command)

Command to change file or directory permissions.

$ chmod u+x script.sh
# Adds execute permission for the owner

$ chmod g-w file.txt
# Removes write permission for the group

$ chmod o=r file.txt
# Sets other's permission to read-only

$ chmod u+rwx,g+rx,o-rwx secret.txt
# Multiple permission changes at once

9. chgrp (Command)

Command to change the group ownership of a file or directory.

$ chgrp developers project.txt
# Changes the group of project.txt to 'developers'

$ ls -l project.txt
-rw-r--r-- 1 john developers 1024 May 10 10:30 project.txt

10. chown (Command)

Command to change the user and/or group ownership of a file or directory.

$ chown alice file.txt
# Changes the owner to 'alice'

$ chown alice:developers file.txt
# Changes both owner to 'alice' and group to 'developers'

$ chown -R alice:developers ./project/
# Recursively changes ownership of a directory and its contents

File Permission Octal +

1. Octal Number Ex: (000)(777)

Permissions can be represented as octal numbers, with three digits for user, group, and others.

$ chmod 755 script.sh
# 7 (rwx) for owner, 5 (r-x) for group, 5 (r-x) for others
$ ls -l script.sh
-rwxr-xr-x 1 john users 1024 May 10 10:30 script.sh

2. R=4

Read permission has a value of 4 in octal notation.

$ chmod 400 private.key
# 4 (r--) for owner, 0 (---) for group, 0 (---) for others
$ ls -l private.key
-r-------- 1 john users 1024 May 10 10:30 private.key

3. W=2

Write permission has a value of 2 in octal notation.

$ chmod 200 temp.txt
# 2 (-w-) for owner, 0 (---) for group, 0 (---) for others
$ ls -l temp.txt
--w------- 1 john users 1024 May 10 10:30 temp.txt

4. X=1

Execute permission has a value of 1 in octal notation.

$ chmod 100 script.sh
# 1 (--x) for owner, 0 (---) for group, 0 (---) for others
$ ls -l script.sh
---x------ 1 john users 1024 May 10 10:30 script.sh

5. ---=0

No permissions has a value of 0 in octal notation.

$ chmod 000 locked.txt
# 0 (---) for owner, 0 (---) for group, 0 (---) for others
$ ls -l locked.txt
---------- 1 john users 1024 May 10 10:30 locked.txt

7. Understanding Umask

Default permission mask that is subtracted from the maximum permissions when creating new files and directories.

$ umask
0022

# With umask 022:
# New files: 666 - 022 = 644 (rw-r--r--)
# New directories: 777 - 022 = 755 (rwxr-xr-x)

# Change umask temporarily
$ umask 027
# Now new files will be 640 (rw-r-----) and directories 750 (rwxr-x---)

# Set permanent umask in ~/.bashrc
$ echo "umask 027" >> ~/.bashrc
Advanced File Permissions +

1. Set SGID

Set Group ID bit makes new files in a directory inherit the directory's group.

$ chmod g+s /shared
$ ls -ld /shared
drwxrwsr-x 2 root developers 4096 May 15 10:30 /shared
# Note the 's' in the group execute position

# Now all new files created in /shared will belong to the 'developers' group

2. Set SUID

Set User ID bit makes a program run with the permissions of the file owner, not the user who runs it.

$ ls -l /usr/bin/passwd
-rwsr-xr-x 1 root root 59640 Mar 22 13:27 /usr/bin/passwd
# Note the 's' in the user execute position

# Set SUID on a file
$ chmod u+s myprogram
$ ls -l myprogram
-rwsr-xr-x 1 root root 12345 May 15 10:30 myprogram

3. Sticky Bit

Sticky bit on a directory prevents users from deleting files they don't own.

$ ls -ld /tmp
drwxrwxrwt 15 root root 4096 May 15 11:45 /tmp
# Note the 't' in the others execute position

# Set sticky bit on a directory
$ chmod +t /shared
$ ls -ld /shared
drwxrwxrwt 2 root developers 4096 May 15 10:30 /shared

4. Remove SGID

Remove the Set Group ID bit from a file or directory.

$ chmod g-s /shared
$ ls -ld /shared
drwxrwxr-x 2 root developers 4096 May 15 10:30 /shared
# The 's' in the group execute position is now gone

5. Remove SUID

Remove the Set User ID bit from a file.

$ chmod u-s myprogram
$ ls -l myprogram
-rwxr-xr-x 1 root root 12345 May 15 10:30 myprogram
# The 's' in the user execute position is now gone

6. Review

Special permissions in octal notation.

# SUID = 4000
# SGID = 2000
# Sticky bit = 1000

# Set all special permissions using octal notation
$ chmod 7755 directory
# 7 = 4 (SUID) + 2 (SGID) + 1 (Sticky bit)
# 755 = rwxr-xr-x

$ ls -ld directory
drwsr-sr-t 2 root root 4096 May 15 10:30 directory

The format of entries in the /etc/shadow file.

# reserved: (reserved field)

DHCP Server +

Introduction to DHCP Server, DHCP Clients

Dynamic Host Configuration Protocol for automatic IP address assignment.

# DHCP (Dynamic Host Configuration Protocol):
# - Automatically assigns IP addresses to clients
# - Provides network configuration (subnet mask, gateway, DNS)
# - Manages lease time for IP addresses
# - Centralizes network configuration management

# DHCP Process:
# 1. DHCP Discover: Client broadcasts request for IP
# 2. DHCP Offer: Server offers an IP address
# 3. DHCP Request: Client requests the offered IP
# 4. DHCP Acknowledge: Server confirms the assignment

1. Install DHCP Server

Installing DHCP server on Linux.

# Install DHCP server on Fedora/RHEL
